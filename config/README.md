# Config

This folder stores repository-level configuration and registries used across the **System Design Template** project.

Its purpose is to keep shared settings, resource locations, and other project-wide references in one predictable place.

## Purpose

Use this folder for configuration data that describes how the repository is organized or where important resources are located.

Typical contents may include:

- external resource registries;
- environment labels;
- feature settings;
- shared application settings;
- lookup tables;
- configuration examples;
- project metadata;
- non-secret connection information.
